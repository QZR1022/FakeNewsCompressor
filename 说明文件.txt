【一、项目概述】

项目名称：FakeNewsCompressor（假新闻监控与智能压缩系统）

一句话描述：
    一个纯C++实现的假新闻检测与选择性压缩归档工具，只有被判定为假新闻
    的文本才会被压缩存储，同时追踪其传播路径，支持关键词检索。

项目定位：
    本作品同时用于参加第二十一届“鼎甲杯”黑框框程序设计大赛，
    以及作为大一软件工程专业的期末作业。

开发周期：20天（5月4日 - 5月24日）
代码规模：2000-3000行 C++ 代码
技术栈：C++17 + STL + Windows.h（零第三方依赖）


【二、项目目标】

2.1 核心目标（必须完成）
    ✅ 从文件读取新闻数据（模拟网络抓包）
    ✅ 对每条新闻进行假新闻检测（朴素贝叶斯分类器）
    ✅ 对假新闻进行 LZ77 压缩归档
    ✅ 追踪每条假新闻的传播次数和来源IP
    ✅ 支持关键词检索已归档的假新闻
    ✅ 提供美观的彩色控制台界面（TUI）

2.2 技术目标（体现水平）
    ✅ 自主实现 LZ77 压缩算法（不调用 zlib 等库）
    ✅ 自主实现朴素贝叶斯分类器（权重自己训练）
    ✅ 模块化设计，高内聚低耦合
    ✅ 使用 Windows.h 实现彩色界面和动态刷新

2.3 比赛目标
    ✅ 完全符合比赛限制（无第三方库、无预训练模型）
    ✅ 代码质量达标（注释清晰、命名规范）
    ✅ 创意分：检测驱动的选择性压缩
    ✅ 难度分：手写压缩算法 + 机器学习模型落地
    ✅ 美观度：专业风格的控制台面板


【三、项目现实意义】

3.1 为什么做这个项目？

    问题背景：
        互联网时代，假新闻传播速度极快。一条假新闻可能在几分钟内
        被转发上万次，造成严重的社会影响。例如：
        - 虚假医疗信息导致公众健康受损
        - 谣言引发恐慌性抢购
        - 政治谣言影响社会稳定

    现有方案的问题：
        - 全量存储所有网络流量 → 磁盘爆炸
        - 只检测不存档 → 无法事后追溯
        - 商业方案成本高、不透明

    本项目的价值：
        ✅ 检测驱动的选择性压缩 → 只存假新闻，节省80%+存储空间
        ✅ 传播链追踪 → 可追溯假新闻来源和扩散路径
        ✅ 离线可用 → 不依赖云端API，数据不外传
        ✅ 完全开源 → 代码透明，可审计

3.2 应用场景（谁可以用？）

    场景1：网信办/舆情监控部门
        - 在临时监控节点部署本系统
        - 实时捕获网络中的可疑信息
        - 压缩后回传中心服务器，节省带宽

    场景2：数字取证/司法鉴定
        - 长期抓包保存传播证据
        - 在法庭上呈现假新闻的完整传播链
        - 压缩存储保证海量数据的可行性

    场景3：个人隐私/家长控制
        - 在家庭网络中监控孩子接触的信息
        - 发现假新闻及时提醒
        - 本地存储，不上传云端

    场景4：学术研究
        - 研究者可收集假新闻传播数据集
        - 用于训练更好的检测模型


【四、项目使用方法】

4.1 编译环境要求
    - Windows 10/11 操作系统
    - Visual Studio 2022（或支持 C++17 的编译器）
    - 无需安装任何第三方库

4.2 编译步骤
    1. 用 Visual Studio 打开项目文件夹
    2. 确保解决方案配置为 "Release" 或 "Debug"
    3. 按 Ctrl + Shift + B 编译
    4. 编译成功后，在输出目录找到 FakeNewsCompressor.exe

4.3 运行前的准备
    - 确保项目根目录下有 data/news.txt 文件
    - news.txt 格式如下（每行一条新闻）：
      
      2026-05-14 10:00:01|192.168.1.1|震惊！林丹决赛前药检阳性！
      2026-05-14 10:00:02|192.168.1.2|中国队夺得金牌，恭喜！
      
    - 格式：时间戳|IP地址|新闻正文（用竖线 | 分隔）

4.4 运行操作说明

    启动程序后，会显示主菜单：
    ┌─────────────────────────────────────────────────┐
    │         FakeNewsCompressor 主菜单               │
    ├─────────────────────────────────────────────────┤
    │  [1] 开始监控 - 读取新闻，检测并压缩假新闻       │
    │  [2] 检索假新闻 - 按关键词搜索已归档的假新闻     │
    │  [3] 查看统计报告 - 压缩率、热点假新闻排行       │
    │  [4] 退出程序                                   │
    └─────────────────────────────────────────────────┘

    选项1 - 监控模式：
        - 程序自动读取 data/news.txt 中的所有新闻
        - 逐条显示检测进度条
        - 每条新闻显示可信度分数（0-100）
        - 假新闻（可信度 < 50）自动压缩保存到 data/archive/
        - 记录该假新闻的传播次数（指纹相同则累加）
        - 监控结束后显示汇总统计

    选项2 - 检索模式：
        - 输入关键词（如："林丹"、"兴奋剂"）
        - 程序返回所有包含该关键词的假新闻
        - 显示新闻内容、可信度、传播次数

    选项3 - 统计模式：
        - 显示总处理新闻数、假新闻数量
        - 显示平均压缩率
        - 显示传播最广的假新闻 Top 5

4.5 输出文件说明
    - data/archive/*.ncz ：压缩后的假新闻文件
    - 压缩文件格式：自定义二进制格式（LZ77 Token 流）


【五、如何把这个项目做成可以上线的产品】

5.1 当前版本（v1.0）的功能边界
    ✅ 从文件读取（模拟抓包）
    ✅ 离线检测（权重预训练）
    ✅ 本地压缩存储
    ✅ 关键词检索
    ❌ 真实网络抓包（未实现）
    ❌ 实时监控（单次运行）
    ❌ 图形界面（控制台版）

5.2 升级到 v2.0（真实抓包版）需要做的
    1. 网络抓包模块：
       - 使用 WinPcap / Npcap 或 Windows raw socket
       - 实时监听网卡流量
       - 提取 HTTP/HTTPS 中的文本内容
    
    2. 实时流处理：
       - 多线程架构（抓包线程 + 检测线程 + 存储线程）
       - 无锁队列传递数据包
       - 内存池减少内存分配
    
    3. 增量训练：
       - 用户可标记误报/漏报
       - 定期重新训练模型更新权重

5.3 升级到 v3.0（云端服务版）需要做的
    1. 后端：
       - 用 C++ 写 REST API 服务（如使用 oatpp 框架）
       - 接入消息队列（如 RabbitMQ）处理大规模流量
       - 用数据库（如 PostgreSQL）存储传播链
    
    2. 前端：
       - Web 控制台（React/Vue）
       - 可视化传播图谱（D3.js）
       - 实时仪表盘
    
    3. 部署：
       - Docker 容器化
       - 云服务器部署（阿里云/腾讯云）
       - 支持分布式抓取节点

5.4 商业化的可能性
    - 面向政府/企业的舆情监控系统
    - 面向媒体的假新闻自查工具
    - 面向教育机构的信息素养教具
    - 开源基础版 + 付费企业版

【九、已知限制与未来改进】

当前版本限制：
    1. 数据来源是文件，不是真实网络抓包
    2. 模型权重是预训练的，不能在线更新
    3. 压缩算法是 LZ77 基础版，未做位流优化
    4. 检索功能是倒排索引，不支持语义搜索
    5. 只支持单线程，未做并发优化

未来改进方向：
    1. 接入 WinPcap 实现真实抓包
    2. 支持在线增量学习更新权重
    3. 压缩算法升级为 LZ77 + Huffman 混合
    4. 接入小型嵌入模型（如 FastText）做语义检索
    5. 多线程流水线架构提高吞吐量
